### A. Tích hợp & Docker-compose hóa (`llm-stack`)
Chúng ta đã gom 2 proxy và 9router cùng Redis vào chung một dự án Docker duy nhất tại `/home/ka/Repos/github.com/trongnghiango/llm-stack`.

- **Tự động Seed dữ liệu cho 9router**:

  - Seed đầy đủ custom provider `CF-AI-PROXY` trỏ qua mạng ảo Docker, 5 Combos (`ka.zzz`, `ka.xxx`, `ka.ddd`, `ka.ccc`, `ka.mmm`), 9 model registry, và password UI mặc định `llmstack2026`.
  - Fix lỗi IP binding của `claude-proxy` (chuyển sang đọc `BIND_ADDR` với mặc định local và `0.0.0.0` trong Docker compose).

### B. Tái cấu trúc logic Tool Call của `cf-ai-proxy`
Giải quyết xung đột về kiến trúc:
- **Loại bỏ `runLocalTool` khi nhận Response từ Cloudflare**: Proxy không tự động chạy Bash/Write/Read nữa và không tự tạo block `tool_result`. Nó chỉ gửi block `tool_use` chuẩn Anthropic về cho client (Claude Code).
- **Bảo toàn Agent Loop**: Claude Code tự thực thi tool trên máy Host và gửi kết quả `tool_result` ở lượt tiếp theo.
- **Giữ nguyên logic convert request**: Proxy chuyển đổi chính xác `tool_result` (Anthropic) thành format OpenAI-compatible `tool` message để Cloudflare LLM hiểu được lịch sử chat.
- Test suite chạy PASS 100%.

1. Đăng nhập vào UI 9router (http://localhost:20128) bằng mật khẩu `llmstack2026`.
