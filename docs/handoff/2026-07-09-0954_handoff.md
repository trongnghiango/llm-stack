## 1. Những việc đã làm được trong phiên này

### A. Tích hợp & Docker-compose hóa (`llm-stack`)
- **Tập trung nguồn sự thật (Source of Truth)**: Hợp nhất toàn bộ mã nguồn của `cf-ai-proxy` và `claude-proxy` về thư mục `services/` trong project `llm-stack`. Thiết lập repo Git tập trung tại dự án này, chấm dứt sửa đổi trên các dự án đơn lẻ cũ.
- **Docker hóa toàn bộ stack**: Tạo `docker-compose.yml` liên kết 4 dịch vụ (`redis`, `cf-ai-proxy`, `9router`, `claude-proxy`) cùng chạy trong mạng ảo `llm-net`.
- **Giải quyết IP Binding**: Chuyển cấu hình listen address của `claude-proxy` từ hardcode `127.0.0.1` sang hỗ trợ biến môi trường `BIND_ADDR` (mặc định local cho host và `0.0.0.0` trong Docker bridge network).
- **Auto-seeding SQLite**: Tạo script `start.sh` tự động hóa quy trình seed dữ liệu SQLite `init.sql` trên máy Host sau khi Next.js container khởi tạo migrations lần đầu. Khắc phục triệt để lỗi mất dữ liệu provider do cơ chế migration của 9router.

### B. Cấu hình Cổng kết nối & Web Monitor
- **Đổi cổng cf-ai-proxy**: Chuyển cổng hoạt động mặc định của `cf-ai-proxy` từ `3000` sang **`20127`** (cập nhật trong `main.go`, `Dockerfile`, `init.sql`, `docker-compose.yml`, và tài liệu hướng dẫn).
- **Expose Web Dashboard**: Expose cổng `20127` của `cf-ai-proxy` ra localhost của Host. Người dùng có thể truy cập trực quan qua đường link [http://localhost:20127/admin/dashboard](http://localhost:20127/admin/dashboard) để theo dõi thời gian thực số lượng Neurons tiêu thụ và lịch sử các Tokens hoạt động.

### C. Tái cấu trúc logic Tool Call
- **Loại bỏ tự thực thi tool ở proxy**: Dọn dẹp hàm `runLocalTool` và block `tool_result` khi model sinh response. `cf-ai-proxy` hiện tại chỉ đóng vai trò phân dịch định dạng (format translation) và trả về block `tool_use` chuẩn Anthropic cho client (Claude Code).
- **Bảo toàn Agent Loop**: Claude Code tự do thực thi local tool (bash, write, update...) dưới máy của bạn và gửi kết quả `tool_result` ở lượt chat kế tiếp. Proxy chuyển đổi ngược lại thành OpenAI-compatible `tool` message để Cloudflare LLM hiểu ngữ cảnh lịch sử.

### D. Tải lại cấu hình tức thì (Hot Reload)
- **Tự động giám sát file CSV**: Cài đặt một background file watcher chạy định kỳ mỗi 5 giây kiểm tra `ModTime` của `accounts.csv` và `models.csv`.
- **Zero-Downtime**: Khi bạn chỉnh sửa trực tiếp các file CSV này trên máy Host, proxy sẽ tự động nạp lại cấu hình tức thì trong nền, không làm gián đoạn hay đứt kết nối HTTP đang chạy, đồng thời tự động kích hoạt đồng bộ Neurons cho các tài khoản mới nạp.
