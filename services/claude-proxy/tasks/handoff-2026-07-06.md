## Những điểm cần lưu ý

### 1. `/debug/metrics` guard ở main.go:49
```go
if r.URL.Path == "/debug/metrics" {
    promhttp.Handler().ServeHTTP(w, r)
    return
}
```
**Không phải dead code** — test gọi `handleProxy()` trực tiếp (không qua mux). Đã giữ lại với comment giải thích. Nếu sau này test chuyển sang dùng server integration test, có thể xoá.

### 2. `applyFlagOverrides()` phải store lại snapshot
Vì CLI flag override chạy sau `Load()`, nó đọc snapshot hiện tại, mutate bản copy, và `globalState.Store()` lại. Nếu quên bước store, flag overrides mất tác dụng.

### 3. `resolveDecision()` giờ trả về `"FALLBACK"` khi unmapped
```go
// 4. Return FALLBACK if no mapping matches — avoids using garbage as model name
return "FALLBACK"
```
Caller trong `routing_fsm.go` check result và fallthrough xuống keyword match nếu `"FALLBACK"`.

### 4. Cycle detection trả về `originalModel`
```go
if visited[currentModel] {
    return originalModel  // thay vì break ambiguous
}
```

### 5. System prompt gửi role riêng
Trước: nhồi system prompt + user task → cùng 1 user message.
Sau:
```go
messages := []map[string]string{
    {"role": "system", "content": systemPrompt},
    {"role": "user", "content": prompt},
}
```
Test `TestCustomModelSettings` đã update để parse messages[0] là system prompt.
