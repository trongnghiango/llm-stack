## 1. Bối cảnh dự án

`cf-ai-proxy` là một reverse proxy viết bằng Go, chạy tại `0.0.0.0:3000`, cho phép:
- Nhận request theo chuẩn **OpenAI** (`/v1/chat/completions`) và **Anthropic** (`/v1/messages`).
- Luân chuyển (failover) qua nhiều tài khoản **Cloudflare Workers AI** để tránh vượt ngưỡng Neurons (10 000 Neurons/ngày/tài khoản).
- Đồng bộ lượng Neurons đã dùng trực tiếp từ **Cloudflare GraphQL API** (Account Analytics: Read).
- Lưu trạng thái Session/Quota vào **Redis** (phân tán).

## 2. Các thay đổi chính trong phiên này

### 2.1 Hỗ trợ Tool Calling (Function Calling)

**Vấn đề ban đầu:**  
Khi dùng Claude Code qua proxy với model `@cf/qwen/qwen2.5-coder-32b-instruct`, các tool như `Write`, `Update` bị gọi nhưng không thực thi được – proxy chỉ forward `tool_use` block về client mà không làm gì.

**Nguyên nhân:**  
- `handleAnthropicStandard` và `handleAnthropicStream` chỉ parse `tool_calls` từ Cloudflare rồi forward về client dưới dạng `tool_use` block.
- Không có cơ chế **thực thi tool cục bộ** (local tool execution).

**Giải pháp đã triển khai:**

#### a) `models.go`
Đã có trường `Tools` và `ToolChoice` trong cả `OpenAIRequest` và `AnthropicRequest` (thêm từ trước).

#### b) `handler.go` – Thêm imports
```go
import (
    "os"
    "path/filepath"
    ...
)
```

#### c) `handler.go` – `handleAnthropicStandard` (non-stream)
Sau khi nhận `tool_calls` từ Cloudflare:
1. Parse `id`, `name`, `argsMap` từ `tool_calls`.
2. Gọi `h.runLocalTool(name, argsMap)`.
3. Trả về **cả hai block**:
   - `tool_use` (để client biết tool đã được gọi)
   - `tool_result` (chứa kết quả thực thi)

#### d) `handler.go` – `handleAnthropicStream` (streaming SSE)
Hai nhánh được cập nhật:
- **Nhánh JSON** (`cfJSON["tool_calls"]`): emit SSE `content_block_start/delta/stop` cho `tool_use`, sau đó gọi `runLocalTool` và emit thêm các events cho `tool_result`.
- **Nhánh XML** (`xmlToolCalls` – dành cho Qwen/DeepSeek): tương tự, dùng index `blkIdx` và `resBlkIdx` để tránh xung đột block index.

#### e) Hàm mới thêm vào cuối `handler.go`

| Hàm | Mô tả |
|-----|-------|
| `runLocalTool(name, args)` | Dispatcher: route theo tên tool sang handler tương ứng |
| `localToolWrite(args)` | `Write`: tạo/ghi đè file. Args: `file_path`, `content` |
| `localToolUpdate(args)` | `Update`: tìm `old_str` và thay bằng `new_str`. Args: `file_path`, `old_str`, `new_str` |
| `localToolRead(args)` | `Read`: đọc nội dung file. Args: `file_path` |

### 2.2 Đồng bộ Neurons từ Cloudflare GraphQL

- Đã thêm **background sync task** trong `session.go` gọi GraphQL API của Cloudflare để lấy số Neurons thực tế đã dùng trong ngày.
- Query sử dụng: `aiInferenceAdaptiveGroups` với `filter.datetime_geq` bằng đầu ngày UTC.
- Kết quả sync log:
  ```
  [Sync - Cloudflare] Đồng bộ thành công a3036b15: 7423 Neurons đã dùng hôm nay
  ```

### Test thực tế (non-stream)
```bash
curl -s -X POST http://localhost:3000/v1/messages \
  -H "Content-Type: application/json" \
  -H "x-api-key: test" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "qwen-2.5-coder",
    "max_tokens": 512,
    "tools": [{
      "name": "Write",
      "description": "Ghi nội dung vào file",
      "input_schema": {
        "type": "object",
        "properties": {
          "file_path": {"type": "string"},
          "content": {"type": "string"}
        },
        "required": ["file_path", "content"]
      }
    }],
    "messages": [{
      "role": "user",
      "content": "Ghi \"Hello from cf-ai-proxy tool!\" vào /tmp/proxy_tool_test.txt"
    }]
  }'
```

**Kết quả:**
```json
{
  "content": [
    {
      "id": "call_1783520541514323051",
      "input": { "content": "Hello from cf-ai-proxy tool!", "file_path": "/tmp/proxy_tool_test.txt" },
      "name": "Write",
      "type": "tool_use"
    },
    {
      "content": "Đã ghi thành công 28 byte vào file: /tmp/proxy_tool_test.txt",
      "tool_use_id": "call_1783520541514323051",
      "type": "tool_result"
    }
  ],
  "stop_reason": "tool_use",
  ...
}
```
